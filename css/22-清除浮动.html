清除浮动

由于父级盒子很多情况下 不方便给高度 但是子盒子浮动又不占有位置 最后父级盒子高度为0 就会影响下面的标准流盒子

清除浮动本质是清除浮动元素造成的影响

clear：left（清除左浮） right（清除右浮） both（全都清除）

清除浮动的策略是闭合浮动（把浮动只闭合在父盒子里）

清除完浮动 父盒子就可以自动检测高度了

清除浮动的方法

1 额外标签法（隔墙法）在最后一个浮动盒子后面写一个块级元素标签 然后给他加一个clear：both的样式

2 父级添加 overflow：hidden（一般用这个）auto scroll （这种方法不会显示溢出父盒子的部分）

3 父级添加 :after伪元素法（直接cv）然后在父盒子的class里面加上classfix（class="box classfix"）

.clearfix:after { content: ""; display: block; height: 0; clear: both; visibility: hidden; } .clearfix {/* IE6、7 专有 */ *zoom: 1; }

相当于用css在后面新增了一个盒子隔开了

4 双伪元素：前后都给他封上（直接cv）后在父盒子的class里面加上classfix（class="box classfix"）

.clearfix:before, .clearfix:after { content: ""; display: table } .clearfix:after { clear:both; } .clearfix{ *zoom: 1; }

css书写顺序

1 布局定位属性（display position float clear visibility overflow）

2 自身属性（width height margin padding border background）

3 文本属性（color font text-decoration text-align vertical-align white-space break-word）

4 其他属性（content cursor border-radius box-shadow background：linear-gradient）

页面布局思路

1 确定页面的版心（可视区）测量可知

2 分析行模块和行模块中的列模块

3 一行中的列模块经常浮动布局 先确定每个列的大小 之后确定列的位置 页面布局第二准则

4 先有结构 再有样式

5 先理清楚布局结构
